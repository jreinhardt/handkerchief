    Ok, maybe we should step back a bit and think about the general approach to this problem first. I am currently not so sure if the current way how handkerchief works is really a good idea. Currently what handkerchief.py does is that it polls the GitHub API for all issues and comments on issues. It then dumps the resulting JSON into the template, where there is a bit of Javascript hackery (see line 49-151) that extracts the infos from JSON data and displays it. So almost everything is done in Javascript. An alternative would be to do much more preprocessing in python. I did not use jQuery or any other JS framework apart from Markdown rendering, because I somehow had the fixed idea to put everything in the python file and wanted to keep dependencies minimal. Now I realized that there is no need to do that, instead one can poll the GitHub API for such content, as handkerchief is only useful when GitHub can be reached anyway. This way I can avoid a lot of the clutter that I have in handkerchief.py right now. I will probably rewrite everything from scratch (which is not a problem, as it is only a few dozen lines of actual code). So maybe the best approach to continue would be if you could mock up a HTML file that looks like you want to have it, and use whatever JS framework or other magic you need for making it awesome. With that I can rewrite handkerchief to obtain and plug in the data in the needed form.

    jreinhardt commented on this issue on 2014-06-06T10:23:06Z
    So, I worked a bit on it, and I think it is coming along nicely. Not yet completely finished, but maybe you want to take a look. I switched to Jinja2 templates, created a template directory, where every subdirectory corresponds to one template. Each subdirectory has to contain a json file named like the subdirectory that contains informations about the template and about which js and css files to inline. This means that it is easy to use arbitrary js libraries. Fort population of the templates, the data directly from the api calls is available for the issues, milestones, comments, labels, repository. For details see here: https://developer.github.com/v3/ The JavaScript part is untouched, i.e. completely broken now. If you want to take a look at that, go ahead, I will probably not have time for that until next week or so. There are a few more things to do for me: * Fetch the templates from github. This allows to use the handkerchief.py completely standalone without installation. * Fetch the avatars and inline them into the html, otherwise they are not available when offline.
